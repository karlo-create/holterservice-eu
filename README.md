# holterservice.eu

Marketing site for **Holterservice** — dr. Obad's Holter EKG distribution and
specialist nalaz service for partner clinics across Croatia.

- Croatian only (`lang="hr"`).
- SEO-first, mobile-first, zero JS by default.
- Designer will iterate visuals against the live preview; visual tokens are
  centralized in [`src/styles/tokens.css`](src/styles/tokens.css). See
  [DESIGN_HANDOFF.md](DESIGN_HANDOFF.md).

## Stack

- **Astro 6** (static output)
- **Tailwind v4** via PostCSS, driven by tokens
- **TypeScript**
- **@astrojs/sitemap** for `sitemap-index.xml`
- Deploys to **Vercel**

## Quick start

```bash
npm install
npm run dev      # http://localhost:4321
npm run build    # static output in dist/
npm run preview  # serve built dist/
```

Requires Node ≥ 22.12.

## SEO

- `sitemap-index.xml` autogenerated by `@astrojs/sitemap`.
- `robots.txt` in `public/`.
- OG + Twitter tags on every page via `src/components/Seo.astro`.
- `MedicalBusiness` JSON-LD on `/`, `LocalBusiness` JSON-LD per partner on
  `/partneri` (empty array OK while `src/data/partners.ts` is empty).
- Canonical URLs derive from `Astro.site` + the current path.

## Forms

`PartnerForm.astro` ships as a **`mailto:`** form for v1 — opens the user's
mail client with a pre-filled body. To upgrade later, swap the `<form action>`
to a POST endpoint and add a server-side recipient. Recipient is documented
in `/privatnost`.

## Performance budget

Targets (Lighthouse mobile, on `/`):

- Performance ≥ 90
- SEO ≥ 95
- Accessibility ≥ 95
- Page weight ≤ 150 KB transferred (HTML + CSS + JS)

The build currently outputs ~12 KB HTML + ~13 KB CSS for the homepage with
zero JavaScript, comfortably under budget.

## Deployment

Vercel auto-detects Astro. No `vercel.json` is required. The build command
is `npm run build` and the output directory is `dist/`. Static-only — no
serverless functions needed for v1.
